#Step 5.4: Set up Systemd for Gunicorn
We want our application to run continuously and start automatically. Create a systemd service file:
sudo nano /etc/systemd/system/myfreshmarket.service

###### CONTENT ######
[Unit]
Description=Gunicorn instance for My Fresh Market app
After=network.target

[Service]
User=ubuntu
Group=www-data
WorkingDirectory=/home/ubuntu/my_fresh_market_app
ExecStart=/home/ubuntu/my_fresh_market_app/venv/bin/gunicorn -w 4 app:app -b 127.0.0.1:8000
Restart=always

[Install]
WantedBy=multi-user.target
##############################
Enable and start the service:
sudo systemctl daemon-reload
sudo systemctl start myfreshmarket
sudo systemctl enable myfreshmarket


CHECK STATUS
sudo systemctl status myfreshmarket

Step 5.5: Configure Nginx as a Reverse Proxy
Nginx will serve as the entry point for web requests and forward them to our Gunicorn app.
sudo nano /etc/nginx/sites-available/myfreshmarket

CONTENT
========
server {
    listen 80;
    server_name YOUR_EC2_PUBLIC_IP; # Or your domain name if you have one

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

##############################
Replace YOUR_EC2_PUBLIC_IP with your instance's public IP address.

Enable this Nginx site and restart Nginx:

sudo ln -s /etc/nginx/sites-available/myfreshmarket /etc/nginx/sites-enabled
sudo nginx -t # Test Nginx configuration for errors
sudo systemctl restart nginx

}
